อธิบายการทำงานเมื่อเน็ตหลุด / เว็บล่ม และวิธีกู้ข้อมูล

ระบบสำรองข้อมูลออฟไลน์อัตโนมัติ (Built-in Active Recovery)

เราได้วางสถาปัตยกรรมระบบเพื่อให้มั่นใจว่า "เช็คชื่อได้ทุกครั้ง แม้ระบบอินเทอร์เน็ตล่ม หรือเว็บแอปเข้าไม่ได้" โดยมีแนวทางดังนี้:

1
กลไกการสแกนใบหน้าแบบ Local:

ระบบประมวลผลใบหน้า (Python AI) ทำงานที่เครื่องของท่าน (เช่น บนบอร์ด Raspberry Pi หรือพีซี) โดยอ้างอิงไฟล์ Cache ใบหน้าถาวร face_encodings_cache.pkl ซึ่งสแกนแบบออฟไลน์ได้ 100% โดยไม่ต้องใช้เน็ต

2
บันทึกประวัติออฟไลน์ลงเครื่อง:

หากระบบหลักหรือหน้าเว็บตัดการทำงาน (HTTP Call ล้มเหลว) โปรแกรม Python GUI จะสลับการทำงานเป็น "Offline mode" อัตโนมัติ และเซฟรายการเช็คชื่อทันทีลงในไฟล์ offline_attendance.json ที่เครื่อง พร้อมแสดงตัวเลขจำนวนคิวที่ค้างรอไว้บนจอ GUI

3
ระบบ Sync ข้อมูลอัตโนมัติ (Auto-Sync Queue):

โปรแกรม Python GUI มี Thread ตรวจสอบสถานะการเชื่อมต่อกับเว็บเซิร์ฟเวอร์แบบเบื้องหลัง ทุกๆ 10 วินาที หากเน็ตกลับมาใช้ได้และเซิร์ฟเวอร์เปิดอยู่ ระบบจะส่งข้อมูลออฟไลน์ขึ้นมาบันทึกที่ Web App และล้างไฟล์แคช offline_attendance.json ออกให้อย่างปลอดภัย

4
การสำรองไฟล์เพื่อย้ายเครื่องหรือติดตั้งใหม่ (Manual Backup):

ท่านสามารถสำรองระบบทั้งหมดได้ง่ายมาก เพียงเก็บโฟลเดอร์เหล่านี้ไว้:

  • โฟลเดอร์ faces/: เก็บรูปถ่ายทั้งหมดที่ใช้เทรน AI
  • โฟลเดอร์ web_app/db.json: ฐานข้อมูลรายชื่อนักเรียน ประวัติเช็คชื่อ และเซสชันทั้งหมด
